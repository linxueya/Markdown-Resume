## <img src="assets/project-diagram-solid.svg" width="30px"> 项目经历

- **配置化商卡**

  *将硬编码商卡迁移为 NoCode 配置平台，支持元素增删、样式与模块级展示控制，显著降低前端重复开发与沟通成本。*

  **原子化与多布局**：按位置与元素拆解为原子组件（标签、图片、文本等），实现页面 / 位置 / 元素三级配置；单套组件覆盖单排、双列、三列布局，控制维护成本。
  **动态样式与事件**：以元素为粒度下发样式与交互；前端预注册事件处理池，运行时按配置完成渲染与绑定，实现展示与交互全动态化。


- **MSC性能优化**

  *针对小象迁移 MSC技术栈 后的启动慢与卡顿问题，通过预拉取、路由预请求与渲染优化等手段提升首屏体验；**秒开率 70.5% → 87.6%**，**冷启 TP90 5492ms → 3126ms**。*

  **启动预拉取**：定位冷启路径上业务接口串行瓶颈；接口合并 + Native 侧预拉取，将请求前置到 APP 启动，小程序侧直接消费预拉取结果，消除冷启串行等待。

  **路由预请求**：热启与 Tab 切换、业务跳转前提前发起请求，与渲染并行，缩短可感知等待。
  **渲染优化**：基于 MSC 基础库插桩与 Perfetto，分析首屏setData频次与同步 API 耗时；按渲染时序拆分首屏关键数据与二屏延迟数据，精简首屏数据量与调用次数。
  **性能稳定性建设**：搭建秒开、预请求成功率与页面启动全链路指标；结合 AB 实验量化单项优化收益。


- **分类页架构演进**

  *解决全量数据下发带来的数据膨胀与反爬风险（竞对单次抓取由 100+ 条降至约 20 条，抓取成本显著上升）；在新零售类小程序分类场景落地流畅的一级分类横滑，改善购物体验。*

  **分页改造**：在一/二/三级分类叠加横滑的高复杂度下，设计以三级分类为 key 的分页结构；各swiper-item独立维护分页状态，支持同 Tab 下多三级列表的独立加载。
  **横滑与列表性能**：预请求并缓存相邻一级分类，切换优先命中缓存；视图层仅保留当前与相邻分类 DOM，离屏销毁控制节点规模；结合可视区与相邻分页的商卡渲染策略，缓解长列表越滑越卡。
